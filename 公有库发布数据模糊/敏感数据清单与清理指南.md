## 二、关键词搜索命令

在添加新代码后，使用以下命令快速检查是否引入了新的敏感数据：

```bash
# 搜索数据库密码（替换为你的真实密码）
findstr /S /I /N "chen030401" *.*

# 搜索旧的 JWT 密钥
findstr /S /I /N "music-edu-jwt-secret-dev" *.*

# 搜索内网 IP
findstr /S /I /N "192.168.1.100" *.*

# 搜索支付宝应用 ID
findstr /S /I /N "ALIPAY_APP_ID=2021" *.*

# 搜索支付宝私钥（PKCS8 格式特征）
findstr /S /I /N "ALIPAY_PRIVATE_KEY=MII" *.*

# 搜索 Tailscale 主机名
findstr /S /I /N "chitamiu.tail" *.*

# 快速检查所有脚本中的硬编码密码
findstr /S /I /N "password.*chen\|PASSWORD.*chen" *.js *.ts
```

## 三、清理操作速查

### 3.1 创建副本后首次清理

```bash
# 1. 全局搜索所有已知敏感关键词
findstr /S /I /N "chen030401\|music-edu-jwt-secret-dev\|192.168.1.100\|ALIPAY_APP_ID=2021\|chitamiu.tail" *.*

# 2. 逐个替换（上述文件清单中的都需检查）
# 3. 提交
git add -A && git commit -m "docs: 替换敏感数据为占位符"
```

### 3.2 更新代码后的快速检查

```bash
# 检查最近修改的文件中是否引入了敏感数据
git diff --name-only HEAD~1..HEAD | xargs findstr "chen030401\|192.168.1.100\|music-edu-jwt-secret\|ALIPAY_APP_ID=2021\|chitamiu.tail"
```

### 3.3 推送前的安全检查清单

- [ ] `backend/.env` 是否不含真实密码/密钥？
- [ ] `backend/src/config/` 中的默认值是否不含内网 IP？
- [ ] `backend/scripts/*.js` 是否不含硬编码数据库密码？
- [ ] `docker-compose.yml` 是否不含支付宝密钥或 Tailscale 主机名默认值？
- [ ] 全局搜索 `chen030401`、`music-edu-jwt-secret-dev`、`192.168.1.100` 是否零结果？
- [ ] 全局搜索 `ALIPAY_APP_ID=2021`、`ALIPAY_PRIVATE_KEY=MII` 是否零结果？
- [ ] 全局搜索 `chitamiu.tail` 是否零结果？

## 四、注意事项

1. **`.env` 文件**：`backend/.env` 已被 Git 跟踪，修改后需重新 git add 才能生效
2. **`.env.example`**：应始终使用占位符值，作为新开发者的配置模板
3. **脚本密码**：所有脚本中的数据库密码使用 `your-db-password-here` 占位符，本地使用时替换为自己的密码
4. **支付宝密钥**：`backend/src/config/alipay.config.ts` 代码本身不含密钥，密钥全部从环境变量读取。但已移除的 `alipay密钥/` 目录曾在旧版历史中存在
5. **`docker-compose.yml` 默认值**：该文件中的 `ALIPAY_*` 和 `TAILSCALE_HOST` 默认值已被替换为占位符，避免容器化部署时不经意泄漏。使用时通过 `.env` 文件或 shell 环境变量传入真实值
6. **Tailscale 主机名**：`docker-compose.yml` 中 `TAILSCALE_HOST` 默认值已替换，如需恢复真实域名请手动设置环境变量
