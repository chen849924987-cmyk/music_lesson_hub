## 一、NestJS 后端（高频：路由顺序 / 列声明 / 类型转换）

| 规则 | 正确做法 | 错误后果 |
|:-----|:---------|:---------|
| **路由顺序** | 具名路由定义在通配路由之前 | 400 `Validation failed` |
| **DTO 类型转换** | `@Type(() => Number)` + `@IsInt()` | 字符串参数校验失败 |
| **联合类型列** | `@Column({ type: 'int', nullable: true })` | 启动崩溃 |
| **手写 SQL 列名** | 用驼峰名（`courseType`），非下划线 | `Unknown column` |
| **JWT payload** | 禁止设 `iat`/`exp` | 签名失败 |
| **@CurrentUser 守卫** | 配对 `OptionalJwtGuard` 或 `AuthGuard('jwt')` | user 始终 undefined |
| **分页额外字段** | 用 `success({ items, meta, stats })` | 额外字段被丢弃 |
| **EntityManager** | 用 `repository.manager` | 模块间不必要依赖 |

## 二、Vue 前端（高频：响应式解构 / 路由 / 文件检查）

| 规则 | 正确做法 | 错误后果 |
|:-----|:---------|:---------|
| **Pinia 解构** | 模板用 `store.xxx`，script 用 `storeToRefs()` | 响应式丢失 |
| **scrollBehavior** | 懒加载组件在 `onMounted` 中处理 hash 滚动 | 滚动失效 |
| **内部导航** | 用 `<router-link>` 替代 `<a>` | SPA 刷新 |
| **路由精确匹配** | 用命名路由 `push({ name })` | 跳转 404 |
| **vue-tsc 变量** | `void refVar` 标记模板使用的变量 | 构建警告 |
| **SFC 末尾** | 必须以 `</style>` 结尾 | 编译报错 |
| **浮点数取整** | `Math.floor(r) % 60` 先取整 | UI 显示小数 |

## 三、价格与金额处理

- **分↔元转换链路**：前端/输入→元，Service→×100→分，数据库→分，Controller→÷100→元
- **规则**：所有价格字段统一处理，前端不感知存储单位

## 四、支付宝支付

- **exec() 返回**：v4.x 直接返回 camelCase 对象（`tradeStatus`），非嵌套结构
- **密钥三件套**：应用私钥/应用公钥/支付宝公钥，缺一不可
- **.env 变更**：必须重启后端生效

## 五、其他

- 前端不感知存储单位
- Toggle 开关即时独立保存
- 响应包装不截断字段
- 种子数据应与产品设计一致
- 各端接口权限分层（公开/教师/管理/制作人）
