# v2.4~v2.8 版本测试指南

> 适用版本：v2.4 ~ v2.8
> 测试范围：教师端运营中心（仪表盘/制作人管理/收益/设置）、审核流程完善、课程播放/试看
> 前置依赖：v2.3 版本测试通过（MinIO 文件存储正常）
> 最后更新：2026-05-13

---

## 1. 环境准备

### 1.1 启动基础设施

```bash
# 启动 MySQL + Redis + MinIO
docker-compose up -d mysql redis minio

# 初始化数据库
node backend/scripts/create-db.js

# 插入种子数据
node backend/scripts/seed.js
```

### 1.2 启动后端

```bash
cd backend
npm run start:dev
# Swagger: http://localhost:3000/api/docs
```

### 1.3 启动前端（可选，本版本主要测试后端 API）

```bash
cd frontend
npm run dev
# http://localhost:5173
```

## 3. API 测试用例

### 3.1 课程审核流程（v2.4/v2.5/v2.7）

#### TC-REV-01 教师提交审核
- **接口：** `POST /api/v1/courses/{id}/submit-review`
- **角色：** 教师 (TEACHER)
- **场景：** 草稿课程 → 提交审核
- **预期：** 状态变为 pending（待审核）

#### TC-REV-02 教师提交审核（已驳回课程）
- **接口：** `POST /api/v1/courses/{id}/submit-review`
- **角色：** 教师 (TEACHER)
- **场景：** 已驳回课程 → 提交审核
- **预期：** 状态变为 pending（待审核）

#### TC-REV-03 教师提交审核（已上架编辑后的课程）
- **接口：** `POST /api/v1/courses/{id}/submit-review`
- **角色：** 教师 (TEACHER)
- **场景：** 已上架课程编辑后 → 提交审核
- **预期：** 状态变为 pending（待审核），pendingEdit 标记被清除

#### TC-REV-04 管理员审核通过课程
- **接口：** `POST /api/v1/courses/{id}/review`
- **角色：** 管理员/审核员 (SUPER_ADMIN/REVIEWER)
- **Body：** `{ "approved": true }`
- **预期：** 状态变为 approved（已上架）

#### TC-REV-05 管理员审核驳回课程
- **接口：** `POST /api/v1/courses/{id}/review`
- **角色：** 管理员/审核员 (SUPER_ADMIN/REVIEWER)
- **Body：** `{ "approved": false, "comment": "课件内容不完整，请补充" }`
- **预期：** 状态变为 rejected，写入 reviewComment

#### TC-REV-06 驳回时缺少原因
- **接口：** `POST /api/v1/courses/{id}/review`
- **角色：** 管理员/审核员 (SUPER_ADMIN/REVIEWER)
- **Body：** `{ "approved": false }`
- **预期：** 返回 400，提示驳回时需填写原因

#### TC-REV-07 教师撤回审核申请（草稿→待审核）
- **接口：** `POST /api/v1/courses/{id}/withdraw-review`
- **角色：** 教师 (TEACHER)
- **场景：** 新创建课程提交审核后撤回
- **预期：** 状态恢复为 draft（草稿）

#### TC-REV-08 教师撤回审核申请（已上架编辑后→待审核）
- **接口：** `POST /api/v1/courses/{id}/withdraw-review`
- **角色：** 教师 (TEACHER)
- **场景：** 已上架课程编辑提交审核后撤回
- **预期：** 状态恢复为 approved（已上架），pendingEdit 标记恢复

#### TC-REV-09 教师申请下架课程
- **接口：** `POST /api/v1/courses/{id}/request-off-shelf`
- **角色：** 教师 (TEACHER)
- **场景：** 已上架课程 → 申请下架
- **预期：** 状态变为 pending（走审核流程），pendingOffShelf=true

#### TC-REV-10 管理员审批下架申请
- **接口：** `POST /api/v1/courses/{id}/review`
- **角色：** 管理员/审核员 (SUPER_ADMIN/REVIEWER)
- **Body：** `{ "approved": true }`
- **场景：** pendingOffShelf=true 的课程（下架申请）
- **预期：** 状态变为 off_shelf（已下架）

#### TC-REV-11 管理员驳回下架申请
- **接口：** `POST /api/v1/courses/{id}/review`
- **角色：** 管理员/审核员 (SUPER_ADMIN/REVIEWER)
- **Body：** `{ "approved": false, "comment": "课程内容合规，暂不下架" }`
- **场景：** pendingOffShelf=true 的课程
- **预期：** 状态恢复为 approved（已上架），pendingOffShelf 清除

#### TC-REV-12 获取待审核课程列表（审核员）
- **接口：** `GET /api/v1/courses/pending-review`
- **角色：** 管理员/审核员 (SUPER_ADMIN/REVIEWER)
- **预期：** 返回 status=pending 的课程列表（分页）

#### TC-REV-13 获取待审核课程数量
- **接口：** `GET /api/v1/courses/stats/pending-count`
- **角色：** 管理员/审核员 (SUPER_ADMIN/REVIEWER)
- **预期：** 返回待审核课程数量

#### TC-REV-14 获取课程审核历史
- **接口：** `GET /api/v1/courses/{id}/reviews`
- **角色：** 管理员/审核员/教师 (SUPER_ADMIN/REVIEWER/TEACHER)
- **预期：** 返回该课程的所有审核记录（按时间倒序）

#### TC-REV-15 学员无权审核课程
- **接口：** `POST /api/v1/courses/{id}/review`
- **角色：** 学员 (STUDENT)
- **预期：** 返回 403

### 3.8 覆盖课程状态流转

#### TC-FLW-01 完整流转：draft → pending → approved
- 创建课程(draft) → 提交审核(pending) → 审核通过(approved)
- **预期：** 每一步状态正确

#### TC-FLW-02 完整流转：draft → pending → rejected → draft → pending
- 创建课程(draft) → 提交审核(pending) → 审核驳回(rejected) → 编辑后重新提交(pending)
- **预期：** 每一步状态正确

#### TC-FLW-03 完整流转：已上架 → 下架申请 → 审核通过 → 已下架
- approved → 申请下架(pending) → 审核通过(off_shelf)
- **预期：** 每一步状态正确

#### TC-FLW-04 完整流转：已上架 → 编辑 → 提交审核 → 审核通过
- approved → 编辑(更新字段) → 提交审核(pending) → 审核通过(approved)
- **预期：** 每一步状态正确

#### TC-FLW-05 非法流转：草稿 → 直接批准
- **接口：** `PATCH /api/v1/courses/{id}/status` 请求 draft → approved
- **预期：** 返回 400，状态流转非法
