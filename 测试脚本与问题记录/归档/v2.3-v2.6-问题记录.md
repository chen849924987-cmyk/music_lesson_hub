## 2026-05-07: UI 设计规范变更与前端端口占用修复

### 问题 1: Vite 端口 5173 被占用导致启动失败
**根因分析：** `vite.config.ts` 中配置了 `strictPort: true`，端口被占用时不尝试其他端口，直接报错退出。
**修复方案：** `frontend/vite.config.ts` - `strictPort: true` → `false`。

### 问题 2: UI 全局字体从 npm 包改为自托管
**变更内容：**
- 新增 `frontend/public/fonts/` SourceSans3-*.woff2（10 个文件）
- 移除 `main.ts` 中 `import 'typeface-source-sans-pro'` 的 npm 包引用
- 移除 `typeface-source-sans-pro` npm 依赖

## 2026-05-07: 页面白屏/仅有淡蓝色背景（卷帘门动画 @keyframes 缺失）

### 问题描述
访问前端页面时，网络请求正常，页面仅显示淡蓝色底色，所有内容完全不可见。

### 根因分析
所有 9 个页面动画 `@keyframes` 定义全局缺失，其中最致命的是卷帘门动画（`curtainSlideLeft`、`curtainSlideRight`、`curtainContentFadeIn`）。

**核心机制：** CSS 规范中，引用未定义的 `@keyframes` 不会触发任何控制台警告或错误。浏览器仅静默跳过该动画属性。

### 修复方案
**文件：** `frontend/src/style.css` - 在文件末尾一次性补全 9 个 `@keyframes` 定义。

## 2026-05-09: 已上架课程重新提交审核的 400 错误

### 问题描述
教师点击编辑已上架课程的「提交审核」按钮时，返回 400 错误：`不允许从 approved 状态变更为 pending 状态`。

### 根因分析
`CoursesService.validateStatusTransition()` 的状态流转表中缺少 `APPROVED → PENDING` 的合法路径。

### 修复方案
**文件：** `backend/src/modules/courses/courses.service.ts`
1. 状态表中添加 `[CourseStatus.APPROVED]: { [CourseStatus.PENDING]: [Role.TEACHER] }`
2. `submitForReview()` 方法增加对 `APPROVED` 状态的支持，并校验需处于 `pendingEdit` 状态。

## 2026-05-09: 教师端下架课程 403 错误

### 问题描述
教师在已上架课程点击「下架」按钮时，返回 403 (Forbidden) 错误。

### 根因分析
`validateStatusTransition()` 中 `APPROVED → OFF_SHELF` 只允许 `SUPER_ADMIN` 和 `REVIEWER` 角色，未授权给 `TEACHER` 角色。

### 业务需求
下架操作需要走审核流程：
1. 教师点击「下架」→ 弹窗确认 → 提交下架申请
2. 管理员审核通过 → 课程下架（`off_shelf`）
3. 管理员审核驳回 → 课程保持已上架（`approved`）

### 修复方案
1. 实体新增 `pendingOffShelf` 标记
2. 新增接口 `POST /api/v1/courses/:id/request-off-shelf` 教师申请下架
3. 审核逻辑修改：`reviewCourse()` 识别下架申请并处理
4. 前端弹窗：点击下架 → 弹窗确认 → 调用下架申请接口

## 2026-05-10: 视频流程相关 Bug 修复

### 问题 1: 后端创建/更新课时时未自动填充视频时长
**根因分析：** `lessons.service.ts` 的 `create()` 和 `update()` 方法未对 `videoId` 对应的视频表进行查询以获取真实时长。
**修复方案：** 注入 `VideoRepository`，创建/更新时自动从 Video 表读取 `duration` 并填充。

### 问题 2: 已驳回课程无法重新提交审核
**根因分析：** 状态流转表 `REJECTED` 只写了 `REJECTED→DRAFT`，缺少 `REJECTED→PENDING`。
**修复方案：** 状态表 `[CourseStatus.REJECTED]` 下新增 `[CourseStatus.PENDING]: [Role.TEACHER]`。

### 问题 3: 编辑课时时视频时长显示为 0
**根因分析：** 编辑弹窗回填时只设置了 `title`、`description`、`isFree` 等字段，未回填 `duration`。
**修复方案：** 编辑弹窗回填时自动设置 `lessonForm.duration = lesson.duration`。

### 问题 4: 教师端课程封面不显示
**根因分析：** 封面 URL 存储在数据库中为 MinIO 对象路径，未转换为可访问的预签名 URL。
**修复方案：** 新增 `transformCoverToUrl()` 方法，所有查询接口返回前调用该方法。

### 问题 5: 单课程编辑页显示系列课 Tab
**根因分析：** `visibleTabs` 是静态定义的，未根据 `courseType` 动态区分。
**修复方案：** `visibleTabs` 改为 `computed`，根据 `courseType` 动态生成。

### 问题 6: 移除视频传 videoId: 0 导致校验失败
**根因分析：** `update()` 方法对所有 `videoId` 都执行视频表查询校验，未识别 `videoId=0` 为"清除关联"的语义。
**修复方案：** `update()` 方法识别 `videoId=0` 为清除关联操作，不触发视频存在性校验。

## 2026-05-10: 附件上传 400 错误 + 系列课课件关联课时 + 后端 502 错误

### 问题 1: 附件上传报 400 "课程ID必须为数字"
**根因分析：** `UploadAttachmentDto` 中使用 `@IsInt()` 校验 `courseId`，但 `multipart/form-data` 传过来的是字符串。缺少 `@Type(() => Number)` 转换器。
**修复方案：** 在 `courseId` 字段上添加 `@Type(() => Number)` 装饰器。

### 问题 2: 系列课课件应与具体课时挂钩，而非挂在系列课程上
**根因分析：** 原有附件系统只支持按课程关联（`courseId`），系列课的课件无法精确关联到具体课时。
**修复方案：** 附件系统新增 `lessonId: number | null` 字段，系列课课时上传课件时指定所属课时。

**涉及后端文件：** attachment.entity.ts, upload-attachment.dto.ts, create-attachment.dto.ts, attachments.service.ts, attachments.controller.ts
**涉及前端文件：** attachment.ts API, AttachmentManager.vue, CourseEdit.vue

### 问题 3: 后端启动 502 Bad Gateway（Attachment.lessonId 列类型不支持）
**根因分析：** TypeORM 的 `@Column()` 装饰器不显式指定 `type`，对于 `number | null` 联合类型，TypeORM 将其推断为 `Object`，而 MySQL 不支持 `Object` 类型。
**修复方案：** `@Column()` → `@Column({ type: 'int', nullable: true })` 显式指定 MySQL 列类型。

## 2026-05-10: 系列课课时 API 404 错误 + 时长输入框冗余

### 问题 1: 系列课添加课时返回 404
**根因分析：** 前端 `saveLesson()` 调用 `POST /courses/:courseId/chapters/:chapterId/lessons`，但后端 `chapters.controller.ts` 的 `@Controller('courses/:courseId/chapters')` 未定义课时子路由。
**修复方案：** `chapters.controller.ts` 注入 `LessonsService`，新增 4 个课时子路由（增删改查）。

### 问题 2: 系列课课时弹窗时长输入框冗余
**根因分析：** 时长需用户手动填写，但视频上传后后端已自动读取真实时长。
**修复方案：** 时长输入框改为 `readonly`，添加提示文案「上传视频后自动从视频文件读取时长」。
