## 2026-05-11: 待审核课程列表 400 错误 + 管理后台侧边栏误高亮 + 课程审核页样式缺失

### 问题 1: 待审核课程列表 API 返回 400 "Validation failed (numeric string is expected)"
**根因分析：** NestJS 中路由定义顺序很重要。`@Get('pending-review')` 方法定义在 `@Get(':id')` 方法之后，导致请求 `/courses/pending-review` 先被 `:id` 参数路由匹配，`ParseIntPipe` 尝试将字符串 `"pending-review"` 解析为数字而失败。
**修复方案：** 将 `@Get('pending-review')` 方法移动到 `@Get(':id')` 方法之前，并注释说明「必须放在 @Get(':id') 之前」。
**经验教训：** NestJS 控制器中具体路径路由必须定义在通配参数路由之前。

### 问题 2: 管理后台侧边栏「课程审核」激活时「课程列表」也高亮
**根因分析：** `AdminLayout.vue` 中导航菜单激活判断使用 `route.path.startsWith(item.path)`，当当前路径为 `/admin/courses/review` 时，因为以 `/admin/courses` 开头，导致两个菜单都满足条件。
**修复方案：** 新增 `isActive(itemPath)` 方法，对 `/admin/courses` 使用**精确匹配**（`===`），其他菜单使用 `startsWith`。

### 问题 3: 课程审核页缺失 alert--warning 样式导致驳回弹窗提示语无样式
**修复方案：** 补充 `.alert` 和 `.alert--warning` 样式定义；优化整体视觉细节。

## 2026-05-11: 已下架课程无法重新提交上架审核

### 问题描述
教师在已下架课程上点击「重新上架」时，前端调用 `POST /api/v1/courses/:id/submit-review` 返回 400 错误。
**根因分析：** `validateStatusTransition()` 状态流转表和 `submitForReview()` 方法均未处理 `OFF_SHELF → PENDING` 的场景。
**修复方案：** 状态表 `OFF_SHELF` 下新增 `PENDING` 流转；`submitForReview()` 允许状态列表增加 `OFF_SHELF`。

## 2026-05-11: 附件审核页 500 错误（Unknown column 'course_type'）

### 问题描述
附件审核页打开后报 500 错误：`Unknown column 'course_type' in 'field list'`。

### 根因分析
`courses` 表的列名是 **`courseType`**（驼峰命名），不是 `course_type`（下划线）。TypeORM 原生查询默认使用 `@Column()` 装饰器中定义的 column name，而 `Course` 实体的 `courseType` 字段没有指定 `name: 'course_type'`，所以 TypeORM 在创建表时使用了字段名 `courseType`。

### 修复方案
`course_type` → `courseType`（SQL 语句和结果读取都做了同步修改）。

### 经验教训
1. **手写SQL列名必须与数据库实际列名一致**：如果 `@Column()` 没有指定 `name` 属性，数据库列名 = TypeScript 属性名（驼峰），不要想当然用下划线。
2. **`getRawMany()` 的字段映射问题**：使用别名时返回的 key 是 `course_courseType` 格式，需手动 `map()` 转换。
3. **`bigint` 字段处理**：`getRawMany()` 返回的 raw 数据中 `bigint` 会被 MySQL2 驱动转为字符串，需要用 `Number()` 转换。
4. **500 错误调试流程**：必须查看后端终端日志才能获取具体错误信息。

## 2026-05-11: 附件审核页"共 0 项"不显示实际总数

### 问题描述
附件审核页卡片表头显示"共 0 项"，但列表实际有数据。
**根因分析：** 后端 `successWithPagination()` 返回 `{ items, meta: { total } }`，但前端使用 `result.total` 取值，应为 `result.meta.total`。
**修复方案：** `total.value = result.meta?.total ?? result.total ?? 0`。
**经验教训：** 所有分页接口应当统一返回 `{ items, meta }` 格式，避免前端每次接入都需要判断。

## 2026-05-12: 制作人端"我的课程"403 Forbidden

### 问题描述
制作人角色访问 `/producer/my-courses` 调用 `/courses/teacher` 返回 403。
**根因分析：** `/courses/teacher` 使用 `@Roles(Role.TEACHER)` 守卫，制作人（PRODUCER）角色不被允许。
**修复方案：** 制作人端改为调用公开接口 `getPublishedCourses`。
**经验教训：** 各端应使用各自权限对应的接口：教师端用 `/courses/teacher`，管理端用 `/courses/admin`，公开端用 `/courses`。

## 2026-05-14: MyOrders.vue / Checkout.vue / OrderManage.vue 文件尾残留 XML 标签

### 问题描述
`MyOrders.vue` 文件末尾意外残留了 `write_to_file` 工具调用时生成的 XML 标签（`</write_to_file>`、`<task_progress>` 等），导致 Vue 编译报错 `Invalid end tag`。

### 根因分析
`write_to_file` 工具的 `content` 参数和工具外部 XML 结构混淆，导致工具自身的 XML 标签被写入文件内容中。

### 影响范围
- 🐛 `MyOrders.vue` → ✅ 已修复
- 🐛 `Cart.vue` → ✅ 已修复
- ✅ `Checkout.vue` → 检查确认干净
- ✅ `OrderManage.vue` → 检查确认干净

### 经验教训
1. Vue SFC 文件末尾不应有非标准标签，Vue 编译器只识别 `<template>`、`<script>`、`<style>` 三种顶层标签。
2. 各页面创建后应检查文件末尾是否以 `</style>` 正确结尾。
