## 1. 环境准备

### 1.1 基础环境

```bash
# 1. 确保 MySQL 已启动
docker-compose up -d mysql

# 2. 初始化数据库
node backend/scripts/create-db.js

# 3. 插入种子数据
node backend/scripts/seed.js
```

### 1.2 MinIO 环境

本版本依赖 MinIO 对象存储服务。请确保 MinIO 已启动：

```bash
# 使用 docker-compose 启动（docker-compose.yml 中需包含 minio 服务，或单独启动）
docker-compose up -d minio

# 或单独启动 MinIO Docker 容器：
docker run -d -p 9000:9000 -p 9001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin123 \
  -v minio_data:/data \
  minio/minio server /data --console-address ":9001"
```

访问 MinIO 控制台：http://localhost:9001 （默认账号：minioadmin / minioadmin123）

### 1.3 后端环境变量配置

确保 `backend/.env` 文件中包含 MinIO 配置：

```env
# MinIO 配置
MINIO_ENDPOINT=localhost
MINIO_PORT=9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin123
MINIO_BUCKET=music-edu
MINIO_USE_SSL=false

# 文件上传限制
UPLOAD_VIDEO_MAX_SIZE=2048
UPLOAD_ATTACHMENT_MAX_SIZE=100
```

### 1.4 启动后端

```bash
cd backend
npm run start:dev
# 启动后访问 http://localhost:3000/api/docs 查看 Swagger 文档
```

## 3. 后端 API 测试（通过 Swagger 或 Python 脚本）

### 3.1 视频管理（Videos）

#### TC-V01 教师上传视频文件
- **接口：** `POST /api/v1/videos/upload`
- **请求方式：** multipart/form-data
- **请求头：** `Authorization: Bearer <teacher01的Token>`
- **请求参数：** `file`（选择一个小视频文件，如 .mp4）
- **预期结果：** 返回 201，包含视频记录的 id、originalName、objectName、fileSize、mimeType、transcodeStatus（pending）等
- **验证点：** transcodeStatus 为 "pending"

#### TC-V02 上传非视频文件
- **接口：** `POST /api/v1/videos/upload`
- **请求头：** `Authorization: Bearer <teacher01的Token>`
- **请求参数：** `file`（上传一个 .txt 或 .jpg 文件）
- **预期结果：** 返回 400，提示"不支持的文件类型"

#### TC-V03 未登录上传视频
- **接口：** `POST /api/v1/videos/upload`
- **请求参数：** `file`（视频文件，无 Token）
- **预期结果：** 返回 401

#### TC-V04 学员上传视频
- **接口：** `POST /api/v1/videos/upload`
- **请求头：** `Authorization: Bearer <学员的Token>`
- **预期结果：** 返回 403

#### TC-V05 教师获取我的视频列表
- **接口：** `GET /api/v1/videos/my`
- **请求头：** `Authorization: Bearer <teacher01的Token>`
- **预期结果：** 返回 200，分页列表，包含刚上传的视频记录
- **验证点：** meta.total >= 1

#### TC-V06 管理员获取视频详情
- **接口：** `GET /api/v1/videos/{刚上传的视频ID}`
- **请求头：** `Authorization: Bearer <admin的Token>`
- **预期结果：** 返回 200，包含视频完整信息

#### TC-V07 审核员获取视频详情
- **接口：** `GET /api/v1/videos/{视频ID}`
- **请求头：** `Authorization: Bearer <reviewer01的Token>`
- **预期结果：** 返回 200

#### TC-V08 学员获取视频详情（无权）
- **接口：** `GET /api/v1/videos/{视频ID}`
- **请求头：** `Authorization: Bearer <学员的Token>`
- **预期结果：** 返回 403（学员不在 TEACHER、REVIEWER、SUPER_ADMIN 角色中）

#### TC-V09 获取视频播放地址
- **接口：** `GET /api/v1/videos/{视频ID}/play-url`
- **请求头：** `Authorization: Bearer <teacher01的Token>`
- **预期结果：** 返回 200，包含临时播放 URL（MinIO 签名 URL，以 http(s):// 开头）

#### TC-V10 学员获取播放地址
- **接口：** `GET /api/v1/videos/{视频ID}/play-url`
- **请求头：** `Authorization: Bearer <学员的Token>`
- **预期结果：** 返回 200（学员有权获取播放地址）

#### TC-V11 获取视频封面地址
- **接口：** `GET /api/v1/videos/{视频ID}/cover-url`
- **请求头：** `Authorization: Bearer <teacher01的Token>`
- **预期结果：** 返回 200，包含封面 URL（如未设置封面，返回空或默认图片 URL）

#### TC-V12 删除视频记录
- **接口：** `DELETE /api/v1/videos/{视频ID}`
- **请求头：** `Authorization: Bearer <teacher01的Token>`
- **预期结果：** 返回 200，提示"删除成功"

#### TC-V13 删除后查询验证

- **请求头：** `Authorization: Bearer <admin的Token>`
- **预期结果：** 返回 404（视频已被软删除）

#### TC-V14 教师删除他人的视频
- **接口：** `DELETE /api/v1/videos/{他人上传的视频ID}`
- **请求头：** `Authorization: Bearer <teacher01的Token>`
- **预期结果：** 返回 404 或 403（仅能删除自己的视频）

#### TC-V15 获取不存在的视频详情

## 6. 常见问题

### Q1: MinIO 连接失败？
- 检查 MinIO 服务是否已启动：`docker ps | grep minio`
- 检查 `.env` 中的 `MINIO_ENDPOINT`、`MINIO_PORT`、`MINIO_ACCESS_KEY`、`MINIO_SECRET_KEY` 配置是否正确
- 尝试使用 MinIO 客户端连接测试：`mc alias set myminio http://localhost:9000 minioadmin minioadmin123`

### Q2: 视频上传接口返回 500？
- 检查 MinIO 存储桶是否已创建（系统会自动创建，但需确认连接正常）
- 检查上传文件大小是否超过限制（默认 2GB）
- 查看后端日志输出，确认具体错误信息

### Q3: 上传文件不支持的类型？
- 视频接口仅支持：mp4、webm、ogg、mov、avi 格式
- 请上传正确格式的视频文件用于测试

### Q4: 获取播放地址/下载地址返回 404？
- 确认视频/附件记录在数据库中存在且未被删除
- 确认 MinIO 中存在对应的文件
- 签名 URL 有过期时间（默认 1 小时），过期后需重新获取

### Q5: 附件审核状态流转规则？
- pending（待审核） → approved（已通过）：审核员操作
- pending（待审核） → rejected（已驳回）：审核员操作
- 已通过/已驳回的附件不能再次审核

### Q6: 没有可用的视频文件用于测试？
- 可以使用 Python 脚本中生成的小型测试视频文件（如 1KB 的 mp4 文件），或使用任意小于 100MB 的 mp4 文件
- 测试脚本中内置了生成假视频文件的功能
