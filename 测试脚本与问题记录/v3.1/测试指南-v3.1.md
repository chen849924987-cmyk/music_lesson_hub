## 1. 环境准备

### 1.1 启动基础设施

```bash
# 启动 MySQL + Redis + MinIO
docker-compose up -d mysql redis minio

# 初始化数据库
node backend/scripts/create-db.js

# 插入种子数据（用户 + 课程）
node backend/scripts/seed.js
node backend/scripts/seed_courses.js
```

### 1.2 启动后端

```bash
cd backend
npm run start:dev
# Swagger 文档: http://localhost:3000/api/docs
```

### 1.3 安装 Python 依赖（自动化测试用）

```bash
pip install requests
```

## 4. API 参考

### 4.1 购物车接口

| 方法 | 路径 | 角色 | 说明 |
|:----:|------|:----:|------|
| GET | `/api/v1/cart` | STUDENT | 获取购物车列表 |
| GET | `/api/v1/cart/count` | STUDENT | 获取购物车数量 |
| POST | `/api/v1/cart/add` | STUDENT | 添加课程到购物车 |
| DELETE | `/api/v1/cart/remove/:courseId` | STUDENT | 移除购物车项 |
| POST | `/api/v1/cart/clear` | STUDENT | 清空购物车 |

**POST `/cart/add` 请求体：**
```json
{
  "courseId": 1
}
```

**GET `/cart` 响应示例：**
```json
{
  "code": 200,
  "message": "操作成功",
  "data": [
    {
      "id": 1,
      "userId": 4,
      "courseId": 1,
      "quantity": 1,
      "course": {
        "id": 1,
        "title": "课程标题",
        "price": 9900,
        "coverImage": "http://..."
      }
    }
  ]
}
```

### 4.2 学员订单接口

| 方法 | 路径 | 角色 | 说明 |
|:----:|------|:----:|------|
| POST | `/api/v1/orders/create` | STUDENT | 创建订单 |
| GET | `/api/v1/orders/my` | STUDENT | 我的订单列表（分页） |
| GET | `/api/v1/orders/:id` | STUDENT/ADMIN | 订单详情 |
| POST | `/api/v1/orders/:id/cancel` | STUDENT | 取消订单 |
| GET | `/api/v1/orders/check-purchase/:courseId` | 已登录 | 检查课程购买状态 |

**POST `/orders/create` 请求体：**
```json
{
  "courseIds": [1, 2, 3]
}
```

**GET `/orders/my` 查询参数：**
- `page`：页码，默认 1
- `pageSize`：每页条数，默认 20
- `status`：按状态筛选（pending/paid/cancelled）

### 4.3 管理端订单接口

| 方法 | 路径 | 角色 | 说明 |
|:----:|------|:----:|------|
| GET | `/api/v1/admin/orders` | SUPER_ADMIN/OPERATOR | 所有订单列表 |
| POST | `/api/v1/admin/orders/:id/refund` | SUPER_ADMIN/OPERATOR | 退款处理 |

**GET `/admin/orders` 查询参数：**
- `page`：页码，默认 1
- `pageSize`：每页条数，默认 20
- `status`：按状态筛选（pending/paid/cancelled/refunded）
- `userId`：按用户ID筛选

## 5. 订单状态流转图

```
pending（待支付）
  ├── 用户取消 → cancelled（已取消）
  ├── 支付成功 → paid（已支付）
  │     └── 管理员退款 → refunded（已退款）
  └── 超时未支付 → expired（已过期）
```

**状态说明：**

| 状态 | 含义 | 可操作 |
|:----:|------|:------:|
| `pending` | 待支付 | 取消、支付 |
| `paid` | 已支付/已完成 | 退款 |
| `cancelled` | 已取消 | 无 |
| `refunded` | 已退款 | 无 |
| `expired` | 已过期 | 无 |

## 7. 注意事项

1. **测试前确保后端已启动**且编译无错误（`npm run start:dev`）
2. **种子数据必须已执行**：`node backend/scripts/seed.js` + `node backend/scripts/seed_courses.js`
3. **课程必须有已上架状态**：确保种子数据中有 `status = approved` 的课程
4. **购物车测试会自动清理**：测试完成后购物车会被清空
5. **临时学员账号自动注册**：用于权限校验的临时账号不会影响种子数据
6. **订单状态测试独立**：每次测试会创建新订单，互不干扰

## 8. 手动测试辅助

如果需要手动验证后端逻辑，可使用以下 curl 命令：

```bash
# 登录学员获取 token
TOKEN=$(curl -s -X POST http://localhost:3000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"student01","password":"admin123"}' | python -c "import sys,json; print(json.load(sys.stdin).get('data',{}).get('accessToken',''))")

# 添加课程到购物车
curl -X POST http://localhost:3000/api/v1/cart/add \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"courseId": 1}'

# 查看购物车
curl http://localhost:3000/api/v1/cart \
  -H "Authorization: Bearer $TOKEN" | python -m json.tool

# 创建订单
curl -X POST http://localhost:3000/api/v1/orders/create \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"courseIds": [1]}' | python -m json.tool

# 查看我的订单
curl "http://localhost:3000/api/v1/orders/my?page=1&pageSize=10" \
  -H "Authorization: Bearer $TOKEN" | python -m json.tool
