## 2026-05-16: 修复学习进度模块 API 的三个 Bug（v4.1）

### Bug 1: duration must be an integer number

**问题描述**：调用 `POST /learning/progress` 时返回 `400 Validation failed (duration must be an integer number)`。

**根因分析**：前端 VideoPlayer 组件从 `videoRef.value.duration` 获取的视频时长为浮点数（如 `186.666667`），但后端 DTO 使用 `@IsInt()` 校验要求整数。

**修复方案**：
| 文件 | 变更 |
|------|------|
| `backend/src/modules/learning/dto/update-progress.dto.ts` | `duration` 字段校验 `@IsInt()` → `@IsNumber()` |

### Bug 2: Field 'userId' doesn't have a default value

**问题描述**：修复 Bug1 后调用接口返回 `500 Field 'userId' doesn't have a default value`。SQL 中 `userId` 被设为 `DEFAULT` 而非实际用户 ID。

**根因分析**：JWT Strategy 的 `validate()` 方法返回的 payload 中用户 ID 存储在 `sub` 字段（来自 `payload.sub`），但 Controller 使用 `@CurrentUser('userId')` 提取，`user.userId` 为 `undefined`。

**修复方案**：
| 文件 | 变更 |
|------|------|
| `backend/src/modules/learning/learning.controller.ts` | 全部 4 个方法的 `@CurrentUser('userId')` → `@CurrentUser('sub')` |

### Bug 3: Duplicate entry for key 'learning_progress.IDX_...'

**问题描述**：修复 Bug1+Bug2 后，偶发报错 `Duplicate entry '155-112' for key...`。

**根因分析**：前端每 30 秒自动同步 + 暂停事件可能在短时间内并发调用 `POST /learning/progress`，两个请求同时发现记录不存在，同时 `INSERT` 相同 `userId+lessonId`，导致唯一索引冲突。

**修复方案**：
| 文件 | 变更 |
|------|------|
| `backend/src/modules/learning/learning.service.ts` | `save()` 创建新记录处添加 `try-catch`，捕获 `ER_DUP_ENTRY` 后自动退化为查找已有记录并更新 |

### 经验教训
1. **JWT payload 字段名**：`passport-jwt` 的 `validate()` 返回对象中的用户 ID 为 `sub` 字段，使用时注意 `@CurrentUser('sub')`。
2. **并发 upsert**：NestJS + TypeORM 中处理并发创建时，需要在 `save()` 外包裹 `try-catch` 兜底唯一索引冲突。
3. **视频时长类型**：浏览器 `HTMLVideoElement.duration` 返回浮点数，后端 DTO 不要用 `@IsInt()` 校验时长字段。

## 2026-05-15: 修复试看倒计时显示小数的问题

**问题描述**：试看模式倒计时显示 `4:59.8` 等小数秒。

**根因分析**：`trialState.remaining` 是浮点数相减结果，`% 60` 未取整。

**修复方案**：
```javascript
const formattedTrialTime = computed(() => {
  const totalSecs = Math.floor(trialState.remaining);
  const mins = Math.floor(totalSecs / 60);
  const secs = totalSecs % 60;
  return `${mins}:${secs.toString().padStart(2, '0')}`;
});
```
