## 1. 环境准备


```bash

# Swagger: http://localhost:3000/api/docs
```


## 4. API 参考

### 4.1 AdminModule 统计

| 方法 | 路径 | 角色 | 说明 |
|:----:|------|:----:|------|
| GET | `/admin/stats` | SUPER_ADMIN | 获取控制台统计数据 |

**响应示例：**
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "totalUsers": 48,
    "totalTeachers": 3,
    "totalCourses": 20,
    "pendingCourses": 2,
    "pendingAttachments": 3
  },
  "timestamp": "2026-05-14T03:00:00.000Z"
}
```

### 4.2 用户管理

| 方法 | 路径 | 角色 | 说明 |
|:----:|------|:----:|------|
| GET | `/users` | SUPER_ADMIN | 用户列表（支持 role/keyword/page/pageSize） |
| PATCH | `/users/:id/toggle` | SUPER_ADMIN | 切换用户启用/禁用 |
| DELETE | `/users/:id` | SUPER_ADMIN | 删除用户 |

**查询参数：**
- `page`：页码，默认 1
- `pageSize`：每页条数，默认 20
- `role`：按角色筛选（admin/teacher/student）
- `keyword`：关键词搜索（用户名/昵称模糊匹配）

### 4.3 教师管理

| 方法 | 路径 | 角色 | 说明 |
|:----:|------|:----:|------|
| GET | `/teachers` | SUPER_ADMIN | 教师列表（支持 page/pageSize） |
| GET | `/teachers/:id` | SUPER_ADMIN | 教师详情 |
| POST | `/teachers/:id/verify` | SUPER_ADMIN | 认证教师 |
| PATCH | `/teachers/:id/review` | SUPER_ADMIN | 审核教师认证 |

**`/teachers/:id/review` 请求体：**
```json
{ "approved": true }   // 通过认证
{ "approved": false }  // 取消认证
```

## 6. 注意事项

1. **测试前确保后端已启动**且编译无错误
2. **种子数据必须已执行**：`node backend/scripts/seed.js`
3. **学员账号自动注册**：测试脚本会注册新学员，不会影响数据
4. **临时用户自动清理**：删除测试会在完成后清除临时用户
5. **测试恢复**：状态切换测试（启用/禁用、认证/取消）会自动恢复原始状态
