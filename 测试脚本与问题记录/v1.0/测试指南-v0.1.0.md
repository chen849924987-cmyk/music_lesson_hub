## 1. 环境准备

### 1.1 启动数据库

```bash
# 确保 MySQL 已启动（本机或 Docker）
# 如果使用 Docker：
docker-compose up -d mysql

# 初始化数据库（如果 music_edu 库不存在）
node backend/scripts/create-db.js

# 插入种子数据（管理员/教师/审核员账号）
node backend/scripts/seed.js
```

### 1.2 启动后端

```bash
cd backend
npm run start:dev
# 启动后访问 http://localhost:3000/api/docs 查看 Swagger 文档
```

### 1.3 启动前端

```bash
cd frontend
npm run dev
# 启动后访问 http://localhost:5173
```

## 3. 后端 API 测试（通过 Swagger）

访问 http://localhost:3000/api/docs，先点击右上角 `Authorize` 按钮输入 Token。

### 3.1 认证模块（Auth）

#### TC-001 学员注册
- **接口：** `POST /api/v1/auth/register`
- **请求体：**
```json
{
  "username": "student_test",
  "password": "test123456",
  "nickname": "测试学员",
  "phone": "13800138000",
  "email": "test@example.com"
}
```
- **预期结果：** 返回 201，包含用户信息（不含密码）
- **验证点：** username 唯一性校验、密码加密存储

#### TC-002 学员登录
- **接口：** `POST /api/v1/auth/login`
- **请求体：**
```json
{
  "username": "student_test",
  "password": "test123456"
}
```
- **预期结果：** 返回 200，包含 `accessToken` 和用户信息
- **验证点：** Token 生成、用户角色为 `student`

#### TC-003 管理员登录
- **接口：** `POST /api/v1/auth/admin/login`
- **请求体：**
```json
{
  "username": "admin",
  "password": "admin123"
}
```
- **预期结果：** 返回 200，包含 `accessToken`
- **验证点：** 学员无法通过此接口登录（会返回 403）

#### TC-004 密码错误登录
- **请求体：** 使用错误的密码
- **预期结果：** 返回 400，`"用户名或密码错误"`
- **验证点：** 错误提示不暴露具体是用户名还是密码错误（安全要求）

#### TC-005 创建教师账号（需 admin Token）
- **接口：** `POST /api/v1/auth/accounts/teacher`
- **请求头：** `Authorization: Bearer <admin的Token>`
- **请求体：**
```json
{
  "username": "teacher02",
  "password": "teacher123",
  "nickname": "李老师"
}
```
- **预期结果：** 返回 201，用户角色为 `teacher`
- **验证点：** 非 super_admin 角色调用返回 403

#### TC-006 创建审核员账号
- **接口：** `POST /api/v1/auth/accounts/reviewer`
- **预期结果：** 返回 201，用户角色为 `reviewer`

#### TC-007 创建运营员账号
- **接口：** `POST /api/v1/auth/accounts/operator`
- **预期结果：** 返回 201，用户角色为 `operator`

### 3.2 用户模块（Users）

#### TC-008 获取当前用户信息
- **接口：** `GET /api/v1/users/profile`
- **请求头：** `Authorization: Bearer <任意登录Token>`
- **预期结果：** 返回 200，包含用户完整信息
- **验证点：** 密码字段不可见

#### TC-009 更新用户信息
- **接口：** `PUT /api/v1/users/profile`
- **请求体：**
```json
{
  "nickname": "新昵称",
  "phone": "13912345678",
  "email": "new@example.com"
}
```
- **预期结果：** 返回 200，信息已更新

#### TC-010 管理员获取用户列表
- **接口：** `GET /api/v1/users?page=1&pageSize=20`
- **请求头：** `Authorization: Bearer <admin的Token>`
- **预期结果：** 返回分页用户列表（含 total, page, pageSize）
- **验证点：** 非管理员角色返回 403

#### TC-011 按角色筛选用户
- **接口：** `GET /api/v1/users?role=teacher`
- **预期结果：** 仅返回角色为 teacher 的用户

#### TC-012 切换用户状态
- **接口：** `PATCH /api/v1/users/2/toggle`（2 为某个用户ID）
- **请求头：** `Authorization: Bearer <admin的Token>`
- **预期结果：** 用户 isActive 字段由 true 变为 false
- **验证点：** 被禁用的用户无法登录

## 7. 常见问题

### Q1: 启动后端报数据库连接错误？
- 确保 MySQL 已启动
- 检查 `.env` 中的 DB_PASSWORD 是否正确
- 先执行 `node backend/scripts/create-db.js` 创建数据库

### Q2: 前端请求后端报跨域错误？
- 确保后端已启动（默认 3000 端口）
- 检查 `backend/src/main.ts` 中的 CORS 配置是否包含前端地址
- 前端默认 5173 端口已在 CORS 白名单中

### Q3: Swagger 接口报 401？
- 需要先在 Swagger 页面右上角点击 `Authorize`
- 输入登录接口返回的 `accessToken`
- 不需要加 "Bearer " 前缀，Swagger 会自动添加

### Q4: 种子脚本报错？
- 确保 `mysql2` 包已安装：`cd backend && npm install`
- 确保 `bcryptjs` 包已安装
- 检查数据库连接信息
