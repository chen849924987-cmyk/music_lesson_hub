## 1. 环境准备

### 1.1 启动数据库

```bash
# 确保 MySQL 已启动（本机或 Docker）
# 如果使用 Docker：
docker-compose up -d mysql

# 初始化数据库（如果 music_edu 库不存在）
node backend/scripts/create-db.js

# 插入种子数据（管理员/教师/审核员账号 + 课程分类 + 示例课程）
node backend/scripts/seed.js
```

### 1.2 启动后端

```bash
cd backend
npm run start:dev
# 启动后访问 http://localhost:3000/api/docs 查看 Swagger 文档
```

### 1.3 启动前端（可选，本版本仅测试后端 API）

```bash
cd frontend
npm run dev
# 启动后访问 http://localhost:5173
```

### 3.1 分类管理（Categories）

#### TC-C01 获取所有分类列表
- **接口：** `GET /api/v1/categories`
- **请求头：** 无需 Token
- **预期结果：** 返回 200，包含 6 个分类
- **验证点：** 返回所有分类（含未启用的）

#### TC-C02 获取启用的分类列表
- **接口：** `GET /api/v1/categories/active`
- **预期结果：** 返回 200，仅包含 isActive=true 的分类
- **验证点：** 默认所有分类均为启用状态

#### TC-C03 获取分类详情
- **接口：** `GET /api/v1/categories/1`
- **预期结果：** 返回 200，包含分类名称"钢琴"、图标"piano"等

#### TC-C04 管理员创建分类
- **接口：** `POST /api/v1/categories`
- **请求头：** `Authorization: Bearer <admin的Token>`
- **请求体：**
```json
{
  "name": "小提琴",
  "icon": "violin",
  "sortOrder": 7,
  "isActive": true
}
```
- **预期结果：** 返回 201，分类创建成功
- **验证点：** 非管理员角色（如 teacher）调用返回 403

#### TC-C05 管理员更新分类

- **请求头：** `Authorization: Bearer <admin的Token>`
- **请求体：**

  "sortOrder": 8

- **预期结果：** 返回 200，分类信息已更新

#### TC-C06 管理员删除分类

- **请求头：** `Authorization: Bearer <admin的Token>`
- **预期结果：** 返回 200，分类删除成功

#### TC-C07 运营员可管理分类

## 6. 常见问题

### Q1: 种子脚本报错 "Table already exists"？
- 先执行 `node backend/scripts/reset-db.js` 清理残留表
- 再执行 `node backend/scripts/seed.js` 重新插入

### Q2: 课程详情接口返回 500？
- 检查数据库连接是否正常
- 检查 TypeORM 实体关系配置是否正确（如 Category、Chapter、Lesson 的关联关系）

### Q3: 创建课程时返回 "分类不存在"？
- 先执行 `GET /api/v1/categories` 确认分类 ID
- 种子数据中分类 ID 从 1 开始（钢琴=1, 吉他=2, 声乐=3, 乐理=4, 古筝=5, 二胡=6）

### Q4: 章节/课时排序不生效？
- 确认请求路径正确：`PATCH /courses/:courseId/chapters/:id/sort`
- 请求体格式：`{ "sortOrder": 1 }`
- 排序权重数值越小越靠前

### Q5: 课程状态流转规则？
- draft（草稿）→ pending（待审核）：教师提交
- pending（待审核）→ approved（已上架）：管理员/审核员审核通过
- pending（待审核）→ rejected（已驳回）：管理员/审核员驳回
- approved（已上架）→ off_shelf（已下架）：管理员下架
- rejected（已驳回）→ draft（草稿）：教师重新编辑
