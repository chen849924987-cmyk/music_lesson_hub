## 1. 系统架构

### 1.1 整体架构

```
用户浏览器
    │
    ▼
┌──────────────────────┐
│  前端 (Nginx :80)    │  ← Vue 3 静态文件
│  /api/* → 后端 :3000 │  ← API 反向代理
└──────┬───────────────┘
       │
       ▼
┌──────────────────────┐
│  后端 (NestJS :3000)  │  ← 业务逻辑
└──┬──┬──┬─────────────┘
   │  │  │
   ▼  ▼  ▼
 MySQL Redis MinIO
```

### 1.2 服务依赖关系

```
frontend ──→ backend ──→ mysql
                        ├── redis
                        └── minio
```

### 1.3 端口分配

| 服务 | 容器端口 | 宿主机默认端口 | 说明 |
|:-----|:---------|:--------------|:-----|
| Nginx（前端） | 80 | 80 | 统一入口 |
| NestJS 后端 | 3000 | 3000 | API 服务 |
| MySQL | 3306 | 3306 | 数据库 |
| Redis | 6379 | 6379 | 缓存 |
| MinIO API | 9000 | 9000 | 对象存储 |
| MinIO Console | 9001 | 9001 | MinIO 管理后台 |

## 3. 快速开始

### 3.1 克隆项目

```bash
git clone https://github.com/你的账号/你的项目.git
cd 你的项目
```

### 3.2 配置环境变量

```bash
# 复制环境变量模板
cp backend/.env.example backend/.env

# 编辑 .env，修改关键配置：
# - DB_PASSWORD: 数据库密码
# - JWT_SECRET: JWT 签名密钥（必须修改！）
# - MINIO_SECRET_KEY: MinIO 密码
```

### 3.3 启动所有服务

```bash
# 一键启动（后台运行）
docker compose up -d

# 查看启动状态
docker compose ps

# 查看实时日志
docker compose logs -f

# 等待所有服务健康检查通过（约 30 秒）
```

### 3.4 初始化数据

```bash
# 创建测试账号（管理员、教师、学员）
docker compose exec backend node dist/scripts/seeds/create-accounts.js

# 如果需要创建测试课程数据
docker compose exec backend node dist/scripts/seeds/seed_courses.js
```

### 3.5 访问验证

```bash
# 前端页面
http://localhost:80

# 后端 API
http://localhost:80/api/v1

# 健康检查
http://localhost:80/health

# Swagger 文档
http://localhost:80/api/docs

# MinIO 管理后台
http://localhost:9001
```

## 4. 环境变量配置

### 4.1 关键环境变量

以下变量**必须**在生产环境中修改，不要使用默认值：

| 变量名 | 默认值 | 说明 | 是否必须修改 |
|:-------|:-------|:-----|:------------|
| `DB_PASSWORD` | `root123456` | 数据库密码 | ✅ 是 |
| `JWT_SECRET` | `your-jwt-secret-key...` | JWT 签名密钥 | ✅ 是 |
| `MINIO_SECRET_KEY` | `minioadmin` | MinIO 密码 | ✅ 是 |
| `REDIS_PASSWORD` | `redis123456` | Redis 密码 | ✅ 是 |
| `ALIPAY_APP_ID` | - | 支付宝应用 ID | ⚠️ 如需支付 |
| `ALIPAY_PRIVATE_KEY` | - | 应用私钥 | ⚠️ 如需支付 |
| `ALIPAY_PUBLIC_KEY` | - | 支付宝公钥 | ⚠️ 如需支付 |

### 4.2 通过 .env 文件配置

在项目根目录创建 `.env` 文件：

```bash
# 数据库
DB_PASSWORD=你的复杂密码
DB_DATABASE=music_edu

# Redis
REDIS_PASSWORD=你的复杂密码

# JWT
JWT_SECRET=生成一个随机字符串，至少32位

# MinIO
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=你的复杂密码

# 支付宝（如需支付功能）
ALIPAY_APP_ID=你的支付宝应用ID
ALIPAY_PRIVATE_KEY=你的应用私钥
ALIPAY_PUBLIC_KEY=你的支付宝公钥
# 通知回调地址（需要公网可访问）
ALIPAY_NOTIFY_URL=https://你的域名/api/v1/payments/notify
ALIPAY_RETURN_URL=https://你的域名/payment/success
```

### 4.3 环境变量优先级

```
.env 文件  <  docker-compose.yml 中的 environment
```

`docker-compose.yml` 中的 `environment` 字段优先级更高，会覆盖 `.env` 中的同名变量。

## 5. Nginx 配置说明

### 5.1 配置位置

Nginx 配置位于 `frontend/nginx.conf`，它被包含在前端 Dockerfile 中。

### 5.2 核心功能

| 功能 | 配置项 | 说明 |
|:-----|:-------|:-----|
| SPA 路由 | `try_files $uri $uri/ /index.html` | 所有路径都返回 index.html |
| API 代理 | `proxy_pass http://backend:3000` | 转发 /api/* 到后端 |
| 大文件上传 | `client_max_body_size 2048M` | 支持最大 2GB 视频上传 |
| Gzip 压缩 | `gzip on` | 压缩静态资源 |
| 静态缓存 | `expires 7d` | JS/CSS 缓存 7 天 |

### 5.3 自定义域名

部署到服务器后，需要修改 Nginx 配置中的 `server_name`：

```nginx
server {
    listen 80;
    server_name www.你的域名.com;  # ← 修改为你的域名
    
    # ... 其他配置保持不变
}
```

配置 HTTPS 时，需要修改为：

```nginx
server {
    listen 443 ssl;
    server_name www.你的域名.com;
    
    ssl_certificate /etc/nginx/certs/fullchain.pem;
    ssl_certificate_key /etc/nginx/certs/privkey.pem;
    
    # ... 其他配置保持不变
}
```

## 6. 数据持久化

### 6.1 数据卷映射

Docker 容器的数据默认会随容器删除而丢失。本项目通过 `volumes` 实现数据持久化：

```yaml
volumes:
  mysql_data:     # MySQL 数据 → 保存在宿主机
  redis_data:     # Redis 数据 → 保存在宿主机
  minio_data:     # MinIO 文件 → 保存在宿主机
  backend_logs:   # 后端日志 → 保存在宿主机
```

### 6.2 数据卷位置

在 Linux 上，Docker 数据卷默认存储在：
```
/var/lib/docker/volumes/
```

可以查看数据卷详情：
```bash
# 列出所有数据卷
docker volume ls

# 查看数据卷详情
docker volume inspect music-edu_mysql_data

# 查看数据卷实际存储位置
docker volume inspect music-edu_mysql_data | grep Mountpoint
```

### 6.3 映射到宿主机目录（可选）

如果你希望数据保存在指定目录（而不是 Docker 默认位置），可以修改 `docker-compose.yml`：

```yaml
volumes:
  # 方式一：使用数据卷（推荐）
  - mysql_data:/var/lib/mysql
  
  # 方式二：直接映射到宿主机目录（更直观）
  # - ./data/mysql:/var/lib/mysql
```

## 7. 常用运维命令

```bash
# 启动所有服务
docker compose up -d

# 启动指定服务
docker compose up -d mysql redis

docker compose down
```


## 8. 日志管理

### 8.1 日志自动清理

Docker 容器的日志会无限增长，需要配置自动清理。

在 `docker-compose.yml` 中添加日志配置：

```yaml
services:
  backend:
    # ... 其他配置
    logging:
      driver: "json-file"
      options:
        max-size: "10m"     # 单个日志文件最大 10MB
        max-file: "3"       # 最多保留 3 个文件
        compress: "true"    # 压缩旧日志
```

### 8.2 手动清理日志

```bash
# 查看容器日志大小
du -sh /var/lib/docker/containers/*/*-json.log

# 清理所有日志
docker compose logs --tail=0

# 彻底清理（需要重启容器）
sudo truncate -s 0 /var/lib/docker/containers/*/*-json.log
```

## 9. 备份与恢复

### 9.1 MySQL 备份

```bash
# 备份整个数据库
docker compose exec -T mysql \
  mysqldump -u root -p${DB_PASSWORD} music_edu > backup_$(date +%Y%m%d).sql

# 备份单个表
docker compose exec -T mysql \
  mysqldump -u root -p${DB_PASSWORD} music_edu users > backup_users.sql

# 恢复数据库
cat backup.sql | docker compose exec -T mysql \
  mysql -u root -p${DB_PASSWORD} music_edu
```

### 9.2 MinIO 文件备份

```bash
# 使用 MinIO Client (mc) 备份
docker compose exec minio mc alias set local http://localhost:9000 minioadmin 你的密码

# 同步文件到本地
docker compose exec minio mc mirror local/music-edu /backup/minio

# 从本地恢复
docker compose exec minio mc mirror /backup/minio local/music-edu
```

### 9.3 自动化备份脚本

创建 `scripts/backup.sh`：

```bash
#!/bin/bash
# 自动备份脚本 - 每天凌晨 3 点运行

BACKUP_DIR="/backup/$(date +%Y%m%d)"
mkdir -p $BACKUP_DIR

# 备份数据库
docker compose exec -T mysql \
  mysqldump -u root -p${DB_PASSWORD} music_edu \
  | gzip > $BACKUP_DIR/music_edu.sql.gz

# 删除 30 天前的备份
find /backup -type d -mtime +30 -exec rm -rf {} \;

echo "备份完成: $BACKUP_DIR"
```

设置定时任务（crontab）：
```bash
# 每天早上 3 点执行备份
0 3 * * * cd /root/项目 && bash scripts/backup.sh
```

## 10. 面试演示指南

### 10.1 方案一：本机开发 + Tailscale Funnel（推荐）

这个方案不需要买服务器，只需要在你的开发电脑上操作：

```bash
# 步骤 1：确保项目在本机能正常运行
# （MySQL/Redis/MinIO 已经启动，npm run dev 可以跑）

# 步骤 2：启动后端
cd backend && npm run start:dev

# 步骤 3：新开终端，启动前端
cd frontend && npm run dev

# 步骤 4：打开浏览器确认 http://localhost:5173 正常运行

# 步骤 5：使用 Tailscale Funnel 暴露前端
tailscale funnel 5173

# 输出类似：https://你的设备名.ts.net
# 把这个网址发给面试官即可！
```

### 10.2 方案二：Docker + Tailscale Funnel（更安全）

```bash
# 步骤 1：确保已安装 Docker

# 步骤 2：构建并启动所有服务
docker compose up -d --build

# 步骤 3：确认服务正常运行
curl http://localhost/health

# 步骤 4：使用 Tailscale Funnel 暴露 80 端口
tailscale funnel 80

# 输出类似：https://你的设备名.ts.net
# 面试官访问这个地址即可体验完整功能
```

### 10.3 面试演示注意事项

- ✅ 提前半小时启动服务，确保一切正常
- ✅ 准备好测试账号（学员/教师/管理员）
- ✅ 准备一些测试数据（课程、评价等）
- ✅ 确保电脑不要休眠
- ✅ 网络要稳定
- ❌ 不要用 5173/3000 端口让面试官访问（端口号不专业）
- ❌ 不要暴露 MySQL/MinIO 端口

## 11. 常见问题排查

### Q1: 启动时报 "port already in use"

```bash
# 查看哪个进程占用了端口
netstat -ano | findstr :80    # Windows
lsof -i :80                    # Mac/Linux

# 停止占用端口的进程，或修改 docker-compose.yml 中的端口映射
```

### Q2: 后端连不上数据库

```bash
# 检查 MySQL 是否正常
docker compose ps mysql

# 检查连接配置
docker compose exec backend env | grep DB_

# 手动测试连接
docker compose exec backend sh
apk add mysql-client
mysql -h mysql -u root -p
```

### Q3: 构建镜像很慢

- 确保 `.dockerignore` 正确配置（排除 `node_modules`）
- 利用 Docker 的层缓存：先复制 `package.json`，安装依赖，再复制源码
- 第一次构建会慢，后续修改代码只影响最后几层，会很快

### Q4: 前端无法访问后端 API

```bash
# 检查 Nginx 配置是否正确
docker compose exec frontend cat /etc/nginx/conf.d/default.conf

# 检查后端是否正常运行
curl http://backend:3000/health

# 检查 Nginx 中 proxy_pass 的地址是否正确
# 应该是 http://backend:3000（使用服务名）
```

### Q5: 视频上传失败

```bash
# 1. 检查 Nginx 的上传大小限制
grep client_max_body_size frontend/nginx.conf

# 2. 检查 MinIO 是否正常
docker compose ps minio

# 3. 检查 MinIO bucket 是否存在
docker compose exec minio mc ls local/music-edu
```

### Q6: 如何迁移服务器？

```bash
# 1. 在新服务器安装 Docker
# 2. 从 Git 拉取项目
# 3. 复制 .env 配置
# 4. 从旧服务器复制数据备份
# 5. 启动服务
docker compose up -d

# 6. 恢复数据
cat backup.sql | docker compose exec -T mysql \
  mysql -u root -p${DB_PASSWORD} music_edu

# 搞定！
```
