## 1. 产品概述

### 1.1 产品定位
一个专注于音乐教育的视频课程在线销售平台，定位为**个人音乐教学品牌网站**（少数教师授课），支持**单课程**和**系列课程**两种售卖模式，实现**管理端、教师端、制作人端（客户端）**三权分立的后台管理体系。

### 1.2 产品目标
- 为音乐教育者提供课程创作与售卖的平台
- 为制作人（学员）提供高质量的音乐视频课程学习体验
- 为平台运营方提供完善的审核、统计与管理工具

### 1.3 核心差异化
- **双课程类型**：单课程（独立售卖）和系列课程（章节式结构化学习）
- **课时级试看**：支持课程级默认试看时长 + 课时级覆盖，秒级控制
- **材料审核分离**：课程主体与附件材料（PDF/乐谱）独立审核，保证资料合规
- **公开课程门户**：首页即课程展示门户，所有访客可浏览课程，无需登录

## 2. 角色权限体系

### 2.1 角色定义

| 角色 | 标识 | 核心职责 |
|------|------|---------|
| **超级管理员** | `super_admin` | 系统全局配置、管理员账号管理、审计日志 |
| **内容审核员** | `reviewer` | 教师入驻审核、课程上架审核、课程资料审核 |
| **运营管理员** | `operator` | 优惠券分发、数据统计查看、订单管理 |
| **教师** | `teacher` | 课程创作与管理、查看制作人数据、收益提现 |
| **制作人（客户端用户）** | `producer` | 浏览课程、购买课程、观看视频、学习进度管理 |

### 2.2 账号管理说明
- **教师账号**：采用**后台录入模式**，由管理员在后台创建教师账号
- **制作人账号**：制作人自行注册（手机号/邮箱注册）
- **内容审核员/运营管理员**：由超级管理员分配

## 4. 非功能性需求

### 4.1 性能要求
- 页面加载时间 < 2s（首屏）
- 视频播放流畅，支持自适应码率
- 并发支持：初期目标支持 500 并发用户

### 4.2 安全要求
- 用户密码加密存储（bcrypt）
- JWT Token 鉴权
- API 接口权限校验（RBAC）
- 视频防盗链（URL 签名 + 过期时间）
- 防 SQL 注入、XSS、CSRF 攻击
- 操作日志审计（15天以上日志自动清理）

### 4.3 可用性要求
- 系统可用性 ≥ 99.9%
- 数据库定时备份
- 文件存储冗余备份

## 5. 业务流程

### 5.1 课程上架流程
```
教师创建课程（单课/系列课）→ 填写信息 → 上传封面
  → 创建章节（系列课）→ 创建课时 → 上传视频（每个课时）
  → 上传附件（课程资料，可选）→ 提交审核
  → 内容审核员审核 → 通过 → 课程上架
                    → 驳回 → 教师修改后重新提交（或撤回）
```

### 5.2 购课流程
```
制作人浏览课程（首页/列表/搜索/分类）→ 查看课程详情 → 试看视频
  → 决定购买 → 选择购买方式（单课/系列/捆绑）
  → 加入购物车（可选）→ 使用优惠券（可选）
  → 生成订单 → 支付宝支付 → 支付成功
  → 课程解锁 → 开始学习
```

### 5.3 学习流程
```
制作人进入制作人中心 → 打开已购课程 → 选择课时
  → 视频播放器加载（带学习进度）
  → 自动记录观看进度 → 下次续播
  → 完成课时 → 跳转下一课时
  → 可下载课程资料（附件）
  → 完成课程后评价
```

### 5.4 教师提现流程
```
教师查看收益概览 → 申请提现（输入金额）
  → 系统校验（最低提现额度）→ 提交申请
  → 管理员审核 → 审核通过 → 线下打款
               → 驳回 → 显示驳回原因
  → 状态更新（已打款）
```
