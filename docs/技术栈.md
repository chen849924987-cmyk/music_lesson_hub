## 缓存架构（Redis）

### 实现位置
- **配置**：`backend/src/config/redis.config.ts` — 8 个环境变量配置项
- **服务**：`backend/src/common/redis/redis.service.ts` — 封装 ioredis，提供完整缓存 API
- **模块**：`backend/src/common/redis/redis.module.ts` — 全局 @Global 模块

### 缓存场景

| 场景 | 缓存键模式 | TTL | 失效策略 |
|------|-----------|:---:|---------|
| 公开课程列表 | `course:catalog:{page}:{size}:{cat}:{type}:{keyword}:{sort}` | 300s | 写操作（编辑/审核/推荐）时清除全部 |
| 课程详情 | `course:detail:{id}` | 600s | 课程更新时清除对应键 |

### 优雅降级
- `REDIS_ENABLED=false` 可完全禁用缓存，业务不受影响
- Redis 连接失败时自动降级（最大重试 3 次，间隔 200ms~2s）
- 所有缓存操作有 try-catch 保护，不会因 Redis 异常导致接口 500

## 响应式设计

### 移动端适配策略
- **断点阈值**：768px（`max-width: 767.98px`）
- **三端布局组件**：AdminLayout、TeacherLayout、ProducerLayout
- **全局样式**：`style.css` 包含响应式 CSS 变量、排版缩放、卡片/按钮/表格移动端适配
- **侧边栏**：桌面端固定240px侧边栏 → 移动端左侧滑入式抽屉（带遮罩层+关闭按钮）
- **顶部栏**：桌面端56px → 移动端48px；"回到主页"和欢迎语在移动端隐藏
- **主内容**：桌面端1.5rem内边距 → 移动端0.75rem
- **公开页面**：首页、课程中心、课程详情、登录/注册页均支持移动端适配
  - 导航栏精简（只显示关键链接）
  - 网格布局单列化
  - 字体、间距、按钮尺寸按比例缩小
- **桌面端（≥768px）** 零视觉变化，移动端（<768px）自动切换紧凑布局

## 角色权限设计

本系统采用 **RBAC（基于角色的访问控制）** 权限模型，共 5 种角色：

| 角色 | 标识 | 说明 |
|------|:----:|------|
| 🛡️ 超级管理员 | `super_admin` | 平台最高权限，可管理所有模块 |
| 🔍 审核员 | `reviewer` | 课程/附件审核操作 |
| ⚙️ 运营员 | `operator` | 订单管理、优惠券管理、数据统计 |
| 👨‍🏫 教师 | `teacher` | 管理自己课程、视频、学员、收益 |
| 🎛️ 制作人 | `producer` | 课程浏览、购买、学习（原"学员"） |
