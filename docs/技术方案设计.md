### 1.2 架构说明

- **前后端分离**：Vue 3 前端应用通过 RESTful API 与 NestJS 后端通信
- **RBAC 权限体系**：基于角色的访问控制，通过 JWT Token 携带角色信息
- **模块化设计**：后端采用 NestJS 模块化架构，每个业务领域独立模块
- **三层架构**：每个模块遵循 Controller → Service → Repository 的分层模式

## 6. 认证与权限体系

### 6.1 认证流程

```
用户登录 → 服务端验证凭据 → 生成 JWT Token
  → Token 中包含: { userId, role, username, exp }
  → 客户端存储 Token (localStorage)
  → 每次请求携带 Authorization: Bearer <token>
```

### 6.2 权限控制

使用 NestJS 的 Guards 机制实现 RBAC：

```typescript
// 角色枚举
enum Role {
  SUPER_ADMIN = 'super_admin',
  REVIEWER = 'reviewer',
  OPERATOR = 'operator',
  TEACHER = 'teacher',
  STUDENT = 'student',
}

// 使用示例
@Post()
@Roles(Role.TEACHER)  // 仅教师可访问
@UseGuards(JwtAuthGuard, RolesGuard)
createCourse() {}
```

### 6.3 JWT Token 结构

```json
{
  "sub": "用户ID",
  "username": "用户名",
  "role": "角色标识",
  "iat": 1234567890,
  "exp": 1234567890
}
```

### 6.4 三端登录隔离

- **学员端**：通过 `/auth/register` 注册，`/auth/login` 登录
- **管理端**：通过 `/auth/admin-login` 登录（仅管理员/审核员）
- **教师端**：通过 `/auth/login` 登录（携带教师角色标识），前端路由守卫根据角色跳转

### 8.2 视频播放流程

```

  → 生成 MinIO 临时访问链接（带签名+过期时间）
  → 返回播放地址 → 前端视频播放器加载播放
```

### 8.3 试看功能实现（已完成）

#### 试看时长机制
- **课程级默认**：Course entity 的 `previewDuration` 字段（单位：**秒**，范围 1~600，默认 300）
- **课时级覆盖**：Lesson entity 的 `previewDuration` 字段（秒级，为 0 时使用课程级设置）
- **isFree 互斥**：设为免费课时时 `previewDuration` 不生效，显示"免费"标识

#### 试看链接生成
- `GET /videos/:id/preview-url` 接口返回签名的临时访问 URL（v2.7 实现）
- 签名 URL 有效期 = `previewDuration` + 60 秒缓冲
- 前端播放器根据 `previewDuration` 限制播放进度条，达到时长后自动暂停并提示"试看结束，请购买完整课程"

#### 防绕过机制
- 签名 URL 过期后自动失效（MinIO 侧控制）
- 后端在校验时不依赖前端传参，直接读取数据库中的 `previewDuration`

- **客户端** (`/*`)：公共页面，展示课程、购课、学习

- **教师端** (`/teacher/*`)：教师工作台布局，侧边栏 + 内容区

每个端有独立的 Layout 组件，通过 Vue Router 的嵌套路由实现。
